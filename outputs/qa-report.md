## 상세 분석

### BUG-001: 퀴즈 연산자 제한

**위치**: `generateProblem()` 함수 (라인 384-401)

**현재 코드**:
```javascript
const operators = ['+', '-'];
```

**문제점**: 초등 저학년 대상으로 덧셈/뺄셈만 의도한 것일 수 있으나, Acceptance Criteria에서 "사칙연산" 퀴즈를 명시함

**권장 수정**:
```javascript
const operators = ['+', '-', '*', '/'];
```
단, 곱셈/나눗셈 추가 시 난이도 조절 필요 (예: 구구단 범위 내)

### BUG-002: 외부 폰트 의존성

**위치**: `<head>` 섹션 (라인 7)

**현재 코드**:
```html
<link href="https://fonts.googleapis.com/css2?family=Jua&display=swap" rel="stylesheet">
```

**권장 수정**: 폰트를 base64로 임베드하거나 시스템 폰트 fallback 강화

## 긍정적 사항

1. **견고한 계산기 로직**: 부동소수점 오류 방지(반올림), 0 나눗셈 예외 처리
2. **우수한 UX**: 연속 계산 지원, Enter 키로 퀴즈 답변 제출 가능
3. **아동 친화적 디자인**: 밝은 색상, 큰 버튼, 격려 메시지
4. **반응형 요소**: 버튼 hover/active 상태, 피드백 애니메이션
